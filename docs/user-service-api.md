# User Service API Spec

Base path:

- Kubernetes ingress: `/users`
- Direct service access: `http://localhost:8001`

## Endpoints

### `GET /health`

Service health check.

Response `200`:

```json
{
  "status": "ok",
  "service": "user-service"
}
```

### `GET /db/health`

Database connectivity check.

Response `200`:

```json
{
  "status": "ok",
  "service": "user-service",
  "db": "ok"
}
```

### `POST /users`

Create a user.

Request body:

```json
{
  "username": "vincent",
  "email": "vincent@example.com"
}
```

Response `201`:

```json
{
  "user_id": "2f04c4a4-d1f8-4d2d-a619-6eec3ab8adbb",
  "username": "vincent",
  "email": "vincent@example.com",
  "created_at": "2026-05-15T10:00:00.000000",
  "updated_at": "2026-05-15T10:00:00.000000"
}
```

Errors:

- `409`: email already exists
- `500`: database or internal failure

### `GET /users`

List users.

Query parameters:

- `limit` default `100`, range `1..500`
- `offset` default `0`

Example:

```http
GET /users?limit=20&offset=0
```

Response `200`:

```json
{
  "items": [
    {
      "user_id": "2f04c4a4-d1f8-4d2d-a619-6eec3ab8adbb",
      "username": "vincent",
      "email": "vincent@example.com",
      "created_at": "2026-05-15T10:00:00.000000",
      "updated_at": "2026-05-15T10:00:00.000000"
    }
  ],
  "limit": 20,
  "offset": 0
}
```

### `GET /users/{user_id}`

Fetch one user by UUID.

Response `200`:

```json
{
  "user_id": "2f04c4a4-d1f8-4d2d-a619-6eec3ab8adbb",
  "username": "vincent",
  "email": "vincent@example.com",
  "created_at": "2026-05-15T10:00:00.000000",
  "updated_at": "2026-05-15T10:00:00.000000"
}
```

Errors:

- `404`: user not found

### `PATCH /users/{user_id}`

Partially update a user.

Request body:

```json
{
  "username": "vincent-lin",
  "email": "vincent.lin@example.com"
}
```

Rules:

- at least one field must be present
- `updated_at` is refreshed automatically

Response `200`:

```json
{
  "user_id": "2f04c4a4-d1f8-4d2d-a619-6eec3ab8adbb",
  "username": "vincent-lin",
  "email": "vincent.lin@example.com",
  "created_at": "2026-05-15T10:00:00.000000",
  "updated_at": "2026-05-15T10:05:00.000000"
}
```

Errors:

- `400`: no fields to update
- `404`: user not found
- `409`: email already exists

### `DELETE /users/{user_id}`

Delete a user by UUID.

Response `204` with empty body.

Errors:

- `404`: user not found

## Notes

- `email` is unique at the database layer.
- UUIDs are generated by PostgreSQL with `gen_random_uuid()`.
- The current implementation uses plain `PATCH` semantics and does not expose `PUT`.
